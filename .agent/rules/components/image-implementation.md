# Image & Icon Implementation Guidelines

> 디자인 구현 시 이미지와 아이콘의 영역 및 경로 처리에 대한 핵심 지침입니다.

## 🔴 핵심 원칙

**"영역(Area)은 코딩하고, 리소스(Source)는 사용자에게 위임한다"**

피그마 디자인을 코드로 구현할 때:

1.  **영역 확보**: 컨테이너와 이미지의 크기(`width`, `height`)는 CSS로 정확히 구현합니다.
2.  **경로 위임**: 실제 이미지 경로(`src`)는 사용자가 직접 설정하거나 수정할 수 있도록 구성합니다.
3.  **구조 유지**: 이미지가 없더라도 레이아웃이 깨지지 않도록 컨테이너 스타일을 견고하게 작성합니다.

## 구현 가이드

### 1. HTML 구조 (Structure)

- 이미지/아이콘을 감싸는 컨테이너 클래스(예: `.img-box`, `.img-frame`)를 반드시 사용합니다.
- `src` 속성에는 명확한 플레이스홀더 경로를 사용하거나, 사용자가 수정해야 함을 알 수 있는 주석을 남깁니다.

```html
<!-- ✅ 권장 패턴 -->
<div class="img-box">
  <!-- [사용자 수정 필요] 실제 아이콘 경로로 변경해주세요 -->
  <img src="../../assets/images/ico_customer_support.svg" alt="고객지원 아이콘" />
</div>
```

### 2. 스타일링 (SCSS)

- 컨테이너에 피그마상의 정확한 너비와 높이를 지정합니다.
- 내부 `img` 태그는 컨테이너에 맞춰 표시되도록 스타일링합니다.

```scss
/* ✅ 권장 스타일 */
.img-box {
  /* 피그마의 정확한 사이즈 적용 */
  width: 80px;
  height: 80px;

  /* 중앙 정렬 및 배치 */
  display: flex;
  align-items: center;
  justify-content: center;

  img {
    /* 이미지가 컨테이너를 벗어나지 않도록 제어 */
    max-width: 100%;
    max-height: 100%;
    object-fit: contain;
  }
}
```

### 3. 경로 처리 (Path Handling)

- **금지**: 존재하지 않는 경로를 마치 있는 것처럼 숨겨서 작성하지 않습니다.
- **금지**: 아이콘/이미지 파일이 없다고 해서 임의로 **SVG 파일을 생성하거나 더미 파일을 만들지 않습니다.** (코드로 파일을 생성하는 행위 금지)
- **금지**: 아이콘 생성 금지
- **금지**: 이미지 생성 금지

- **권장**:
  - 다운로드가 실패하거나 파일이 없는 경우에도 **영역(Area)** 코드는 완벽하게 작성합니다.
  - 사용자는 `src`만 수정하면 즉시 디자인 대로 보일 수 있어야 합니다.

- **필수**:
  - 아이콘은 사용자가 따로 다운로드 받아서 사용하도록 합니다.
